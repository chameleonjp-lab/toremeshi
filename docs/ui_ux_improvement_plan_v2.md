## 3. ロール別レビュー（主要指摘）

### 良好と確認できた点（現状維持）
- 読み込み: 6本の `fetch` を `Promise.all` + `AbortSignal.timeout(12s)` + `cache:'force-cache'`、失敗時は再読込ボタン、スピナーあり（堅牢）。
- a11y: `:focus-visible` リング、`prefers-reduced-motion`、`#srStatus` の `aria-live`、正誤の記号＋テキスト併記（色依存解消済）。
- 3D演出: 遅延ロード＋`body.playing` 連動表示＋空箱装飾抑制。失敗時もゲーム継続（`safe()`）。
- 二重加算: `state!=='Consulting'` ガードで多重タップ安全。

### 課題（ロール別）
- **④日本語コピー/識字 / ⑤相談者情報**（最重要）: 100問すべてで惜しい/不正解の理由文が**同一の汎用文**。さらに `userReason()` が `主条件`→`今回の相談` と置換し「一部は合うが、**今回の相談が足りない**。」という**非文**を生成。学び（仕様3.1の核心価値）が伝わらない。
- **②UX/オンボーディング**: スタート直後に世界観・役割の説明がなく、いきなりカウントダウン→出題。プレイヤーが「自分は何者で、何をする人か」を掴めない。
- **①ビジュアル / トーン**: タイトル副題「15人の筋トレ相談に答えろ」は命令調で、ゲーム本編の親切なトーンと差がある（ブランドとして許容範囲）。
- **照合**: README のデータ記載が `v1` のままで実態（`v2`）と不一致。

## 5. 採用項目の実装方針（index.html）

### 5-1. フィードバック解説の具体化（#2/#3）
- `userReason()` の置換語を `主条件`→**`大切な条件`** に変更（非文の解消。他の置換は維持）。
- `choiceNutrients(q,id)` を新設: `itemById`（読込済の栄養データ）から、その問題タイプで重視する栄養素値を整形（`allergen_tags` は生タグ表示を避け除外、数値のみ）。
- `explainChoice(q,c)` を新設: 正解は従来の `makeCorrectReason(q)` を返し、惜しい/不正解は「（役割の理由文）＋『この候補は、たんぱく質が◯g…です。』」を返す**追記方式**。`renderFeedback` の「ひとこと解説」と「ほかの選択肢」リストの両方で使用。
- 読みやすさ統一のため、`makeCorrectReason()` の数値併記からも `allergen_tags`（生の英語タグ `contains_soy` 等）を除外し、栄養数値のみ併記する（正解/惜しい/不正解で表記をそろえる）。
- これは **表示のみの拡充**。役割（正誤）はデータの値をそのまま用い、再判定しない。

### 5-2. ホーム微修正（#4）
- ホーム説明文「あなたはジムの**トレーナー**。…」→「あなたはジムの**新人トレーナー**。…」。

### 5-3. README 整合（#5）
- データファイル節に `questions_v2_100.csv`（相談文形式・現行ロード対象）を明記し、実装状況に一文追記。

## 6. 【新規画面】世界観イントロ画面 — 追加要件

### 6-1. 目的
スタート直後に「プレイヤーの役割（ジムの新人トレーナー）」「これから起きること（相談者が来る）」「やること（相談に合う食事を教える）」を、親切なトーンで1画面で提示し、オンボーディングを補強する。

### 6-2. 画面遷移への組み込み
```
タイトル/名前入力(Home)
  ↓ [スタート]（名前必須・localStorage保存・deck生成）
【新規】世界観イントロ画面(Intro)   ← 本計画で追加
  ↓ [相談を受ける]
3.2.1 カウントダウン
  ↓
相談1/15 …（以降は従来どおり）
```
- 実装上は `startGame()` 末尾の `countdown(3)` を `renderIntro()` 呼び出しに置換。`renderIntro()` の「相談を受ける」で `countdown(3)` を起動。
- deck 生成（`makeDeck`）は従来どおり `startGame()` 内で先に実行（生成失敗時は従来の `renderErrors` が優先表示され、イントロに到達しない）。

### 6-3. 表示内容（コピー）
見出し「はじめに」＋本文4行。依頼テキストを**日本語として正しい形に微修正**して掲載する（識字性・正確性の観点。意味は不変）:

| 行 | 掲載文（修正後） | 依頼原文との差分 |
|---|---|---|
| 1 | あなたはジムの新人トレーナーです。 | 変更なし |
| 2 | あなたの元に相談者がやってきます。 | 変更なし |
| 3 | どうやらみなさん、筋トレ後の食事についてお悩みの様子です。 | 「お悩み様子」→「お悩みの様子です」＋読点付与 |
| 4 | 相談内容に合わせて、最適な食事を教えてあげましょう。 | 読点付与のみ |

ボタン: 主「相談を受ける」（全幅・先頭）／副「名前を入れ直す」（→ Home へ戻る）。

### 6-4. 状態・操作・アクセシビリティ
- 新状態 `state='Intro'`。`setPlaying(false)`（3Dは非表示）、`topbar` は隠す、`panel` は `home` レイアウト（中央寄せ）。
- 描画後 `.app` をトップへスクロール（`toTop()`）、主ボタンへ `focus({preventScroll:true})`（Enter/Space で進行可）。
- `#srStatus` に役割と次操作を読み上げ（`say(...)`）。
- 「名前を入れ直す」で `renderHome()` に戻れる脱出導線を用意（操作性）。

### 6-5. ビジュアル
- 既存パネルに溶け込む `.story`（縦並び・行間広め `line-height:1.85`・`clamp(16px,4.4vw,19px)`）。
- 320px幅でも横スクロールが出ないこと。主従が明確なボタン配置（主=全幅）。

### 6-6. 受け入れ基準
- スタート後にイントロが必ず1回表示され、「相談を受ける」でカウントダウン→出題へ進む。
- 「名前を入れ直す」で Home に戻れる。
- 15問・スコア・ランキング・3D表示の挙動は従来どおり（回帰なし）。
- `prefers-reduced-motion` 環境でも破綻しない。
