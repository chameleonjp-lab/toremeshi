# トレメシ UI/UX/操作性 改善 実装計画書 v1

作成日: 2026-06-19
対象: `index.html`（単一ファイル実装のゲーム本体）
方式: Opus による5ロール並列レビュー（ビジュアル/UI・UX/オンボーディング・アクセシビリティ・パフォーマンス/読込・QA/操作テスト）→ 統合 → 最適案選定 → 実装。

## 0. 大前提（厳守する不変条件）

次は**変更しない**:
- クイズの正誤判定（`answer` の `c.role`）、スコア計算（速度・連続ボーナス）
- 問題データ / CSV・JSON 構造 / `parseCSV` / `validate` / デッキ生成（`makeDeck`/`arrange`/`pickFor`）
- Supabase 連携（`submit_score` / `get_best_score_ranking`、送信 payload）

実装範囲は **見た目 / UI / UX / 操作性 / 読み込み体験 / アクセシビリティ** に限定し、すべて `index.html` 内（`<style>`・`<script>`・最小の DOM）で完結させる。`three-client.js` / `three-client.css` は今回は変更しない（読み込みタイミングのみ index.html 側で制御）。

## 1. レビュー結果サマリ（ロール別）

各ロールが行番号付きで指摘（実ブラウザ実行は不可のため、コード精読＋全画面メンタル操作テスト＋ヒューリスティクスで評価）。良好と確認された点も多い:
- **良好**: システムフォント採用、`answer()` の二重加算ガードは単一スレッド前提で安全、`.choice.near_miss` スタイルは実在・適用、全角空白名は `trim` で弾かれる、3D は `safe()` で例外隔離されゲーム停止しない、ダーク地のテキストコントラストは概ね 9:1 以上で良好。
- **課題の集中領域**: ①3D演出パネルの常時表示と無条件ロード、②キーボード/フォーカス可視性、③正誤の色依存、④読み込み/エラーの堅牢性、⑤画面遷移時のスクロール・フィードバック欠如。

詳細な指摘ID（V1–V10 / U1–U12 / A1–A12 / P1–P10 / Q1–Q12）は本リポジトリのレビュー記録に基づく。

## 2. 採用項目と実装方針（index.html）

### グループA: 3D演出の統合最適化（V1/P1/P5/P7/Q1/Q5）
- **A-1 遅延ロード**: 静的 `<script src="./three-client.js">` を撤去し、`startGame()` で `import('./three-client.js')` を一度だけ実行。ゲームを始めるまで Three.js 一式（CDN）を取得しない。
- **A-2 表示の状態連動**: `#three-character-root` を既定 `display:none`、`body.playing` のときだけ表示。`renderTop()`（=対戦中の Approaching/Consulting/AnswerFeedback）で `playing` を付与、Home/Countdown/Result/Loading/Error で外す。→ Home/結果での巨大空パネル、リプレイ後のポーズ残留（Q1）が解消。
- **A-3 空時の装飾抑制**: `#three-character-container:empty` の枠線/影/背景を無効化し、ロード前/失敗時の「壊れた空箱」を防ぐ。

### グループB: アクセシビリティ中核（A1/A2/A4/A5/V2/V10/U2）
- **B-1**: viewport から `maximum-scale=1, user-scalable=no` を削除（WCAG 1.4.4）。
- **B-2**: 全インタラクティブ要素に `:focus-visible` リングと `:active` 押下フィードバックを付与。
- **B-3**: `@media (prefers-reduced-motion:reduce)` でアニメーション/トランジションを無効化、`backdrop-filter` を切る。
- **B-4**: 正誤表示を `outline` から `box-shadow` リング＋淡い面色へ（角丸追従・非テキストコントラスト改善）。さらに「ほかの選択肢」ボタンに `✓正解 / △惜しい / ✗不正解` のテキスト＋記号を併記（色のみ依存を解消）。
- **B-5**: 視覚非表示の `aria-live` ステータス領域（`#srStatus`）を追加し、カウントダウン開始・各相談・結果を読み上げ。`#countdown` は `aria-hidden`。

### グループC: ビジュアル整え（V3/V6/V7/V9/U8/U10）
- **C-1**: `.panel h1/h2/h3` と `.big` のスケール定義（クラス無し見出しを整える）。
- **C-2**: `details>summary` のアフォーダンス（カーソル/開閉マーカー）。
- **C-3**: カウントダウンを `min(28vw,42vh)` で縦溢れ防止＋出現アニメ（reduced-motion 時は無効）。
- **C-4**: 結果アクションを `.result-actions` 化し、主アクション「もう一度プレイ」を全幅・先頭に（主従明確化）。外部リンクの inline スタイルを CSS へ移管。
- **C-5**: `.next-action` の過大な下マージン（100px）を縮小。

### グループD: UX/操作フロー（U1/U3/U4/U5/U9/Q2/Q3）
- **D-1**: 名前入力で Enter→スタート（`enterKeyHint='go'`）。
- **D-2**: フィードバック描画後に前進ボタンへ `focus({preventScroll:true})`（Enter/Space で次へ）。
- **D-3**: 画面遷移（renderGame/renderFeedback/renderResult）で `.app` をトップへスクロール。
- **D-4**: Home に遊び方3点（全15問/速さ・連続でスコア/「惜しい」の意味）を追記。
- **D-5**: トップバーに進捗バー（`(idx+1)/15`）。
- **D-6**: シェアにトースト（コピー成功/失敗/共有）を表示。

### グループE: 読み込み堅牢性（P2/P3/P4/P9/U12/Q4）
- **E-1**: 起動6 fetch に `AbortSignal.timeout(12s)` と `cache:'force-cache'`。電波不良の無限「準備中…」を解消、バージョン付き静的データを再取得しない。
- **E-2**: `loadData()` 冒頭で `rows/itemById/questions/byType` をリセット（再試行で `validate()` の「ID重複」誤検出を防止）。
- **E-3**: エラー画面をユーザー向け文言＋「再読み込み」ボタンにし、開発用テーブルは `<details>` 折りたたみ。
- **E-4**: ローディングにスピナー（reduced-motion で停止）。
- **E-5**: ランキングが「送信成功・取得のみ失敗」のとき「再取得」ボタンを表示。

## 4. リグレッション確認チェックリスト

- 初回ロード→Home 表示、名前未入力で開始不可、Enter で開始可。
- カウントダウン→15問完走、正解/惜しい/不正解の点数が従来どおり加算。
- 連続回答・多重タップで二重加算なし（既存ガード維持）。
- 正解=緑/惜しい=黄/不正解=赤 に加え記号・テキストが出る。
- 結果画面のスコア・統計・ランキング送信が従来どおり。
- 3Dキャラはゲーム中のみ表示、Home/結果では非表示。読み込み失敗でもゲームは続行。
- スマホ幅で UI が重ならない。`prefers-reduced-motion` で演出が抑制される。
- データ取得失敗時に「再読み込み」で復帰できる。

## 5. ロールバック

すべて `index.html` 内の追加・上書きで、対象コミットを revert すれば完全に元へ戻る。ロジック（判定/スコア/データ/Supabase）には未介入のため、表示・操作面のみの影響に限定される。
